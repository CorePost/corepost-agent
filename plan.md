# Plan for corepost-agent

## Роль репозитория

Post-boot агент на Linux: `systemd + curl`, периодически проверяет статус устройства и применяет политику отзыва доверия после загрузки ОС.

## Текущее состояние

- Репозиторий создан заново, реализации пока нет.
- Агент обязателен, а в текущем дереве кода он полностью отсутствует.

## Что сделать

- Определить контракт агента с сервером:
  - heartbeat;
  - получение статуса/policy;
  - подтверждение применения действия;
  - сценарий восстановления.
- Реализовать агент как systemd service с конфигом и локальным state file.
- Использовать минимальный стек, который вы обозначили: `systemd + curl` плюс утилиты для HMAC/TLS.
- Реализовать реакции по политике:
  - уведомление пользователя;
  - блокировка сессии;
  - завершение пользовательской сессии;
  - shutdown/reboot при жёстком отзыве доверия.
- Предусмотреть поведение при потере сети, backoff и безопасный default mode.
- Добавить journald logging, диагностический режим и минимальный self-check.
- Подготовить Debian VM в QEMU и проверить:
  - что systemd unit стартует на чистой системе;
  - что тулчейн (`curl`, openssl/sha256 tooling, network, journald) реально присутствует;
  - что агент переживает перезагрузку VM;
  - что отзыв доверия реально отрабатывает на живой системе.
- Описать процедуру установки через `corepost-install`.

## Критерий готовности

- На чистой Debian VM агент устанавливается, стартует через systemd и воспроизводимо выполняет post-boot отзыв доверия.
